# 基于 U1 的智能家居环境监测与控制系统 PRD

## 1. 项目定位

本项目基于小米 AIoT U1 开发平台，将现有示例工程改造为一个智能家居环境监测与自动控制系统。系统以 U1 主控子板为核心，连接光照、温湿度、按键、LED、数码管、风扇等功能子板，实现环境数据采集、自动联动控制、手动控制和本地状态显示。

项目当前阶段以本地闭环控制为主：U1 直接读取传感器并控制执行器。C3 Wi-Fi 子板仅作为后续联网和 LLM 自然语言控制的预留接口，本阶段不实现真实云端通信和大模型调用。

## 2. 项目目标

### 2.1 用户目标

用户可以通过该系统完成以下操作：

- 查看当前环境温度和光照强度的简化显示。
- 在自动模式下，让系统根据环境光照自动开关 LED。
- 在自动模式下，让系统根据温度自动开关风扇。
- 通过按键切换自动/手动模式。
- 在手动模式下，通过按键直接控制 LED 和风扇。

### 2.2 工程目标

工程上需要将当前 `demo` 工程从“录音与超声波测距”示例改造为智能家居主业务：

- 保留并复用现有 U1、I2C、S1、E1 等基础代码。
- 新增 S2 光照、S8 温湿度、E2 风扇、C3 预留接口模块。
- 重写 `main.c` 主循环，使其围绕智能家居状态机运行。
- 补齐硬件地址、阈值、采样周期、PWM 占空比等配置项。

## 3. 用户与使用场景

### 3.1 典型使用者

- 课程设计、实验课、嵌入式开发练习的学生或教师。
- 需要演示 AIoT 子板协同工作的开发者。
- 后续希望扩展联网控制、语音控制或 LLM 控制的项目开发者。

### 3.2 典型场景

1. 系统上电后进入自动模式。
2. 用户遮挡光照传感器，环境光照低于阈值，LED 自动打开。
3. 用户恢复正常光照，LED 自动关闭。
4. 环境温度达到阈值后，风扇自动打开。
5. 用户按下 K1 切换到手动模式。
6. 用户按下 K2 手动开关 LED，按下 K3 手动开关风扇。
7. 数码管持续显示温度和简化后的光照值。

## 4. 功能范围

### 4.1 本阶段范围

本阶段必须完成以下功能：

- S2 光照传感器采集。
- S8 温湿度传感器采集，其中温度参与控制，湿度作为扩展数据保留。
- E1 RGB LED 控制。
- E1 四位数码管显示。
- E2 风扇 PWM 控制。
- S1 按键输入。
- 自动/手动模式切换。
- C3 Wi-Fi 接口空桩预留。
- 异常读数和 I2C 失败的基本容错。

### 4.2 非本阶段范围

以下能力不在当前阶段实现：

- Wi-Fi 配网。
- TCP、HTTP、MQTT 等联网通信。
- 云端服务。
- LLM API 调用。
- 语音识别。
- 手机 App 或 Web 控制台。
- 阈值持久化保存到 Flash。

## 5. 硬件组成

| 子板 | 作用 | 核心器件 | 通信方式 | 推荐地址/接口 |
| --- | --- | --- | --- | --- |
| U1 | 主控 | GD32470ZKT6 | I2C / UART / GPIO | 主控，无外设地址 |
| S1 | 按键输入 | HT16K33 | I2C | `0xE8` |
| S2 | 光照采集 | BH1750 | I2C | `0x46` |
| S8 | 温湿度采集 | SHT35 | I2C | `0x88` |
| E1 LED | RGB LED 控制 | PCA9685 | I2C | `0xC0` |
| E1 数码管 | 四位数码管显示 | HT16K33 | I2C | `0xE0` |
| E2 | 风扇控制 | PCA9685 | I2C | `0xC8` |
| C3 | Wi-Fi / BLE 预留 | ESP32 | UART | UART AT 接口 |

推荐统一配置：

```c
#define S2_LIGHT_ADDR        0x46
#define S8_TEMP_HUMI_ADDR    0x88
#define E1_LED_ADDR          0xC0
#define E1_DISPLAY_ADDR      0xE0
#define E2_FAN_ADDR          0xC8
#define S1_KEY_ADDR          0xE8
```

## 6. 功能需求

### 6.1 环境数据采集

系统需要周期性读取环境数据：

- 光照值：由 S2/BH1750 采集，单位为 lx。
- 温度值：由 S8/SHT35 采集，单位为摄氏度。
- 湿度值：由 S8/SHT35 采集，单位为 %RH，本阶段不参与控制。

默认采样和控制周期为 500 ms。

```c
#define SENSOR_UPDATE_MS     500
```

### 6.2 自动模式

系统上电默认进入自动模式。

在自动模式下，系统按以下规则控制执行器：

| 条件 | 动作 |
| --- | --- |
| 光照值 `< LIGHT_THRESHOLD` | 打开 E1 LED |
| 光照值 `>= LIGHT_THRESHOLD` | 关闭 E1 LED |
| 温度值 `>= TEMP_THRESHOLD` | 打开 E2 风扇 |
| 温度值 `< TEMP_THRESHOLD` | 关闭 E2 风扇 |

默认阈值：

```c
#define LIGHT_THRESHOLD      500
#define TEMP_THRESHOLD       28
```

### 6.3 手动模式

用户按下 K1 后，系统在自动模式和手动模式之间切换。

在手动模式下，系统不再根据阈值自动修改 LED 和风扇状态，只响应按键输入：

| 按键 | 功能 |
| --- | --- |
| K1 | 自动/手动模式切换 |
| K2 | LED 开/关切换 |
| K3 | 风扇开/关切换 |

### 6.4 显示功能

E1 四位数码管用于显示温度和简化光照值。

显示规则：

```text
前两位：温度整数值
后两位：光照值 / 100
```

示例：

| 温度 | 光照值 | 显示值 |
| --- | --- | --- |
| 26 摄氏度 | 780 lx | 2607 |
| 28 摄氏度 | 1200 lx | 2812 |
| 30 摄氏度 | 6500 lx | 3065 |

光照显示值需要限制在 0 到 99，避免数码管溢出。

### 6.5 执行器控制

LED 和风扇使用 PCA9685 PWM 控制。

默认控制参数：

```c
#define LED_ON_DUTY          100
#define FAN_ON_DUTY          80
```

关闭状态占空比为 0%。风扇实际启动占空比可根据硬件联调结果在 60% 到 80% 之间调整。

### 6.6 C3 预留接口

本阶段保留 C3 Wi-Fi 子板接口，但不接入真实网络服务。预留接口用于后续扩展：

```c
void Wifi_Init(void);
void Wifi_SendSensorData(float temperature, float humidity, uint16_t light);
void Wifi_ReceiveCommand(void);
```

后续可扩展为：

- 上传温度、湿度、光照和设备状态。
- 接收远程 LED、风扇、模式切换命令。
- 对接后端服务和 LLM 自然语言控制。

## 7. 状态模型

系统维护一个统一状态结构：

```c
typedef enum {
    MODE_AUTO = 0,
    MODE_MANUAL = 1
} SystemMode;

typedef struct {
    float temperature;
    float humidity;
    uint16_t light;
    uint8_t led_on;
    uint8_t fan_on;
    SystemMode mode;
} SmartHomeState;
```

上电默认状态：

- `mode = MODE_AUTO`
- `led_on = 0`
- `fan_on = 0`
- 温湿度和光照使用首次成功采集值

## 8. 错误处理与边界

- I2C 读取失败时，保留上一次有效数据。
- 连续读取失败时，执行器不应频繁抖动。
- 光照值限制在 0 到 65535 lx。
- 温度显示限制在 0 到 99 摄氏度，避免四位数码管显示溢出。
- 光照显示值使用 `light / 100`，并限制在 0 到 99。
- 上电初始化完成前，LED 和风扇保持关闭。

## 9. 验收标准

### 9.1 编译验收

- 工程可以在现有 Keil/VS Code 工程结构中编译。
- 不出现缺失符号、重复定义、头文件找不到等错误。

### 9.2 硬件验收

- S2 能读出随遮光变化的光照值。
- S8 能读出合理温度值。
- E1 LED 能按控制逻辑开关。
- E1 数码管能稳定显示温度和光照缩放值。
- E2 风扇能按控制逻辑开关。
- S1 的 K1/K2/K3 能触发对应动作。

### 9.3 功能验收

- 上电默认自动模式。
- 光照低于 500 lx 时 LED 打开，光照恢复后 LED 关闭。
- 温度达到 28 摄氏度时风扇打开，温度降低后风扇关闭。
- K1 可切换自动/手动模式。
- 手动模式下 K2 控制 LED，K3 控制风扇。
- 手动模式下自动阈值逻辑不抢占用户控制。
