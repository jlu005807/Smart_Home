# S8 温湿度传感器子板整理与使用示例

## 1. S8 子板定位

S8 是传感器子板，主要用于采集环境中的**温度**和**湿度**数据。它集成了 **SHT35-DIS-B 温湿度传感器**，通过温湿度一体式探头进行测量，并通过 **I2C 总线**与 U1 主控通信。PPT 第 2 页说明，S8 子板包含电源指示灯、地址拨码开关和温湿度传感器，其中温度传感器可取出，角度可以灵活调整。

在你的智能家居系统中，S8 主要负责：

```text
采集温度值
    ↓
U1 判断温度是否超过阈值
    ↓
温度高：打开 E2 风扇
温度低：关闭 E2 风扇
    ↓
E1 数码管前两位显示温度
```

---

| 项目     | 内容          |
| ------ | ----------- |
| 子板名称   | S8 温湿度传感器子板 |
| 核心传感器  | SHT35-DIS-B |
| 功能     | 采集温度和湿度     |
| 通信方式   | I2C         |
| 地址选择   | 拨码开关        |
| 电源指示灯  | 1 个         |

PPT 第 3 页的功能框图显示，S8 子板通过对外接口接入 I2C 总线，内部包含地址拨码、电平转换、电源转换和 SHT35 温湿度传感器模块。

## 3. 测量范围与精度

| 测量对象 |            范围 |      精度 |
| ---- | ------------: | ------: |
| 温度   |   -40℃ ~ 125℃ |   ±0.1℃ |
| 湿度   | 0%RH ~ 100%RH | ±1.5%RH |

这些参数来自 PPT 第 4 页的“温湿度传感器使用”部分。

## 4. I2C 地址

S8 的 I2C 地址由拨码开关控制：

```c
#define S8_ADDR_1  0x88
#define S8_ADDR_2  0x8A
```

在本项目中，建议默认使用：

```c
#define S8_TEMP_HUMI_ADDR  0x88
```

注意：这里 PPT 给出的 `0x88 / 0x8A` 很可能是**8 位 I2C 地址**。如果你使用的 GD32 I2C 库要求传入 7 位地址，则可能需要右移一位：

```text
0x88 >> 1 = 0x44
0x8A >> 1 = 0x45
```

所以实际写代码时要参考官方 Demo。如果 Demo 里直接写 `0x88`，你就继续用 `0x88`；如果 Demo 用 `0x44`，就说明库函数使用的是 7 位地址。

---

## 5. S8 常用命令

PPT 第 4 页给出了两个核心命令：

| 命令       | 名称 | 功能        |
| -------- | -- | --------- |
| `0x30A2` | 复位 | 软复位 SHT35 |
| `0x2C0D` | 模式 | 连续读模式     |

使用步骤：

```text
1. 向 S8 地址发送复位命令 0x30A2
2. 等待 200ms
3. 读取数据前，发送连续读模式命令 0x2C0D
4. 等待 100ms
5. 从 S8 地址读取 6 个字节
```

---

## 6. 读取数据格式

S8 一次读取 6 个字节，格式如下：

|  字节序号 | 含义         |
| ----: | ---------- |
| Byte0 | 温度高字节      |
| Byte1 | 温度低字节      |
| Byte2 | 温度 CRC 校验值 |
| Byte3 | 湿度高字节      |
| Byte4 | 湿度低字节      |
| Byte5 | 湿度 CRC 校验值 |

PPT 第 5 页给出了温湿度数据计算方式和 6 字节数据排列顺序。

---

## 7. 数据计算公式

### 7.1 温度计算

```c
temp_raw = (temp_high << 8) + temp_low;
temperature = temp_raw * 175.0 / 65535.0 - 45.0;
```

### 7.2 湿度计算

```c
humi_raw = (humi_high << 8) + humi_low;
humidity = humi_raw * 100.0 / 65535.0;
```

其中：

```text
temp_high = Byte0
temp_low  = Byte1

humi_high = Byte3
humi_low  = Byte4
```

# 8. S8 使用流程

## 8.1 初始化流程

```text
S8 初始化
    ↓
向 S8 发送复位命令 0x30A2
    ↓
延时 200ms
    ↓
初始化完成
```

## 8.2 读取流程

```text
读取 S8 温湿度
    ↓
向 S8 发送连续读模式命令 0x2C0D
    ↓
延时 100ms
    ↓
读取 6 个字节
    ↓
组合温度原始值和湿度原始值
    ↓
根据公式转换为实际温度和湿度
```

# 9. 使用示例一：S8 驱动接口设计

可以把 S8 单独封装成一个驱动模块：

```text
drivers
├── s8_temp_humi.c
└── s8_temp_humi.h
```

## s8_temp_humi.h

```c
#ifndef S8_TEMP_HUMI_H
#define S8_TEMP_HUMI_H

#include <stdint.h>

#define S8_TEMP_HUMI_ADDR    0x88

#define S8_CMD_RESET         0x30A2
#define S8_CMD_READ_MODE     0x2C0D

typedef struct {
    float temperature;
    float humidity;
} S8_Data;

void S8_Init(void);
uint8_t S8_ReadData(S8_Data *data);

#endif
```

---

## s8_temp_humi.c

下面是偏通用的写法，你后续只需要把 `I2C_WriteCmd16()` 和 `I2C_ReadBytes()` 替换成你工程里的 I2C 函数即可。

```c
#include "s8_temp_humi.h"
#include "bsp_i2c.h"
#include "delay.h"

static void S8_WriteCmd(uint16_t cmd)
{
    uint8_t buf[2];

    buf[0] = (uint8_t)(cmd >> 8);
    buf[1] = (uint8_t)(cmd & 0xFF);

    I2C_WriteBytes(S8_TEMP_HUMI_ADDR, buf, 2);
}

void S8_Init(void)
{
    S8_WriteCmd(S8_CMD_RESET);
    Delay_ms(200);
}

uint8_t S8_ReadData(S8_Data *data)
{
    uint8_t buf[6];
    uint16_t temp_raw;
    uint16_t humi_raw;

    if (data == 0)
        return 0;

    S8_WriteCmd(S8_CMD_READ_MODE);
    Delay_ms(100);

    if (I2C_ReadBytes(S8_TEMP_HUMI_ADDR, buf, 6) != 0)
        return 0;

    temp_raw = ((uint16_t)buf[0] << 8) | buf[1];
    humi_raw = ((uint16_t)buf[3] << 8) | buf[4];

    data->temperature = temp_raw * 175.0f / 65535.0f - 45.0f;
    data->humidity = humi_raw * 100.0f / 65535.0f;

    return 1;
}
```

# 10. 使用示例二：在智能家居主循环中使用 S8

在你的项目中，S8 主要用于**温度控制风扇**，示例逻辑如下：

```c
#include "s8_temp_humi.h"
#include "e2_fan.h"
#include "e1_led_display.h"

#define TEMP_THRESHOLD  28.0f

int main(void)
{
    S8_Data env_data;

    System_Init();
    I2C_Init();

    S8_Init();
    E2_Fan_Init();
    E1_Display_Init();

    while (1)
    {
        if (S8_ReadData(&env_data))
        {
            if (env_data.temperature >= TEMP_THRESHOLD)
            {
                E2_Fan_On();
            }
            else
            {
                E2_Fan_Off();
            }

            E1_DisplayTemperature((int)env_data.temperature);
        }

        Delay_ms(500);
    }
}
```

# 11. 使用示例三：结合你的完整系统显示

你的系统中，数码管前两位显示温度，后两位显示光照。S8 负责提供温度部分：

```c
S8_Data env_data;
uint16_t light_lux;
uint8_t temp_display;
uint8_t light_display;

S8_ReadData(&env_data);
light_lux = S2_ReadLight();

temp_display = (uint8_t)env_data.temperature;
light_display = light_lux / 100;

E1_DisplayFourDigits(temp_display / 10,
                     temp_display % 10,
                     light_display / 10,
                     light_display % 10);
```

例如：

```text
温度：26.8℃
光照：1200 lx

temp_display = 26
light_display = 12

数码管显示：2612
```

# 13. 需要特别注意的点

## 13.1 地址问题

PPT 给出的地址是：

```text
0x88 / 0x8A
```

但你需要确认工程里的 I2C 函数使用的是：

```text
8 位地址：直接用 0x88
7 位地址：使用 0x44
```

这是最容易出错的地方。

---

## 13.2 命令是 16 位

SHT35 的命令不是 1 个字节，而是 2 个字节。

例如复位命令：

```text
0x30A2
```

发送时应该拆成：

```c
0x30, 0xA2
```

连续读模式命令：

```text
0x2C0D
```

发送时应该拆成：

```c
0x2C, 0x0D
```

---

## 13.3 读取 6 个字节

读取时必须一次读取 6 个字节：

```text
温度高字节
温度低字节
温度 CRC
湿度高字节
湿度低字节
湿度 CRC
```

当前最小实现可以先不校验 CRC，但如果数据经常异常，建议加 CRC 校验。

---

## 13.4 显示时建议取整数

数码管只有四位，而且你的设计是一半显示温度、一半显示光照，所以温度建议显示整数部分：

```c
int temp_display = (int)temperature;
```

例如：

```text
26.8℃ → 显示 26
27.2℃ → 显示 27
```

如果想四舍五入：

```c
int temp_display = (int)(temperature + 0.5f);
```

# 14. 最简版 S8 使用总结

```text
1. S8 使用 SHT35-DIS-B 温湿度传感器。
2. 通过 I2C 与 U1 通信。
3. I2C 地址由拨码开关决定，为 0x88 或 0x8A。
4. 初始化时发送 0x30A2 复位命令，并等待 200ms。
5. 读取时发送 0x2C0D 连续读模式命令，并等待 100ms。
6. 然后读取 6 个字节。
7. 前两个字节计算温度，第 4、5 个字节计算湿度。
8. 温度公式为：temperature = raw * 175 / 65535 - 45。
9. 湿度公式为：humidity = raw * 100 / 65535。
10. 在本项目中，S8 的温度值用于控制 E2 风扇，并显示在 E1 数码管前两位。
```
