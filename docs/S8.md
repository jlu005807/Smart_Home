# S8 温湿度传感器子板配置与编程说明

## 1. 子板概述

S8 子板是小米 AIoT 实训箱中的 **温湿度传感器子板**，属于传感器类子板。它集成了 **SHT35-DIS-B 温湿度传感器**，可以采集环境温度和湿度数据。子板上主要包含电源指示灯、地址拨码开关和温湿度传感器探头，其中温湿度传感器可取出，角度可以灵活调整。

## 2. 子板硬件组成

| 编号 | 名称     | 数量 | 作用          |
| -- | ------ | -: | ----------- |
| ①  | 电源指示灯  |  1 | 指示子板是否正常供电  |
| ②  | 拨码开关   |  1 | 用于选择 I2C 地址 |
| ③  | 温湿度传感器 |  1 | 采集温度和湿度数据   |

S8 子板使用 **I2C 总线**与主控板通信。功能框图中包含外部接口、I2C 总线、地址拨码、电平转换、电源转换和 SHT35 温湿度传感器模块。子板输入侧为 5V，板上会进行 5V 到 3.3V 的电源转换。

---

## 3. 通信方式

```text
通信协议：I2C
传感器芯片：SHT35-DIS-B
拨码开关数量：1 个
数据类型：温度 + 湿度
读取方式：发送测量命令后读取 6 字节数据
```

Agent 或程序在使用 S8 子板时，应将它识别为：

```yaml
board: S8
type: sensor
protocol: i2c
chip: SHT35-DIS-B
function:
  - temperature
  - humidity
```

## 4. I2C 地址配置

资料中给出的 S8 子板 I2C 地址为：

```text
0x88，0x8A
```

这两个地址由 **1 个拨码开关**控制。

需要注意的是，很多资料中的 I2C 地址是 **8 位地址**，而 Linux `smbus`、Arduino Wire、部分 HAL 库通常使用 **7 位地址**。

转换规则：

```text
7 位地址 = 8 位地址 >> 1
```

因此可整理为：

| 拨码状态 | 资料中的 8 位地址 | 常用 7 位地址 |
| ---- | ---------: | -------: |
| 状态 0 |       0x88 |     0x44 |
| 状态 1 |       0x8A |     0x45 |

注意：PDF 中没有明确说明拨码开关的 **ON/OFF 到底对应状态 0 还是状态 1**。因此实际开发时，建议先运行 I2C 扫描程序确认地址。

Agent 判断规则：

```text
如果用户明确说 S8 地址是 0x88，则 7 位地址为 0x44。
如果用户明确说 S8 地址是 0x8A，则 7 位地址为 0x45。
如果用户没有说明拨码状态，Agent 应提示先扫描 I2C 地址。
扫描时重点检查 0x44 和 0x45。
```

## 5. 测量范围与精度

S8 子板中的温湿度传感器包含温度传感器和湿度传感器。资料中给出的测量参数如下：

| 数据类型 | 测量范围          | 精度      |
| ---- | ------------- | ------- |
| 温度   | -40℃ ~ 125℃   | ±0.1℃   |
| 湿度   | 0%RH ~ 100%RH | ±1.5%RH |

## 6. SHT35 指令配置

S8 子板主要使用两个命令：

|     命令 | 名称 | 功能           |
| -----: | -- | ------------ |
| 0x30A2 | 复位 | 软复位 SHT35    |
| 0x2C0D | 模式 | 连续读模式 / 测量模式 |

资料中的使用流程是：初始化时向传感器发送复位指令并等待 200ms；读取数据时先写入 `0x2C0D` 命令，等待 100ms，然后从传感器地址读取 6 个字节。

---

## 7. 初始化流程

S8 初始化流程如下：

```text
1. 确认 S8 子板已连接到 I2C 总线
2. 确认子板电源指示灯亮起
3. 根据拨码开关确定 I2C 地址
4. 如果不确定地址，扫描 0x44 和 0x45
5. 向 SHT35 发送软复位命令 0x30A2
6. 等待 200ms
7. 初始化完成
```

伪代码：

```c
s8_init(address):
    i2c_write_command(address, 0x30A2)
    delay_ms(200)
```

---

## 8. 数据读取流程

S8 读取温湿度数据的流程：

```text
1. 向 SHT35 地址写入测量命令 0x2C0D
2. 等待 100ms
3. 从 SHT35 地址读取 6 个字节
4. 解析温度高字节和温度低字节
5. 跳过温度 CRC
6. 解析湿度高字节和湿度低字节
7. 跳过湿度 CRC
8. 根据公式计算实际温度和湿度
```

6 个字节的数据格式如下：

|  字节序号 | 含义         |
| ----: | ---------- |
| byte0 | 温度高字节      |
| byte1 | 温度低字节      |
| byte2 | 温度 CRC 校验值 |
| byte3 | 湿度高字节      |
| byte4 | 湿度低字节      |
| byte5 | 湿度 CRC 校验值 |

---

## 9. 数据计算公式

资料中给出的计算方法如下：

```text
温度值 = (温度高字节 << 8) + 温度低字节

温度实际值 = 温度值 * 175 / 65535 - 45

湿度值 = (湿度高字节 << 8) + 湿度低字节

湿度实际值 = 湿度值 * 100 / 65535
```

整理为代码表达：

```c
raw_temp = (temp_high << 8) | temp_low;
temperature = raw_temp * 175.0 / 65535.0 - 45.0;

raw_humi = (humi_high << 8) | humi_low;
humidity = raw_humi * 100.0 / 65535.0;
```

## 10. 推荐封装接口

为了方便 Agent 或主程序调用，建议不要让上层直接操作寄存器，而是封装成如下接口。

### 10.1 初始化接口

```c
int s8_init(uint8_t addr);
```

功能：

```text
初始化 S8 子板，发送软复位命令，等待传感器稳定。
```

### 10.2 读取接口

```c
int s8_read(float *temperature, float *humidity);
```

功能：

```text
读取当前温度和湿度，并转换为实际值。
```

### 10.3 状态判断接口

```c
int s8_get_temperature_level(float temperature);
```

可以约定：

|                      温度范围 | 状态 |
| ------------------------: | -- |
|         temperature < 18℃ | 偏冷 |
| 18℃ <= temperature <= 28℃ | 舒适 |
|         temperature > 28℃ | 偏热 |

## 12. Agent 编程规则

Agent 在生成 S8 相关代码时，必须遵守以下规则：

```text
1. S8 是 I2C 子板，不是 UART 子板。
2. S8 使用 SHT35-DIS-B 温湿度传感器。
3. S8 的资料地址为 0x88 和 0x8A。
4. 如果当前代码库使用 7 位 I2C 地址，应转换为 0x44 和 0x45。
5. 不允许直接猜测拨码开关 ON/OFF 与地址的物理对应关系。
6. 如果用户没有明确给出地址，先建议运行 I2C 扫描。
7. 初始化时先发送 0x30A2 软复位命令，并等待 200ms。
8. 读取数据前发送 0x2C0D 命令，并等待 100ms。
9. 每次读取 6 字节。
10. byte0、byte1 用于计算温度。
11. byte3、byte4 用于计算湿度。
12. byte2、byte5 是 CRC 校验值，基础实验中可先跳过；正式程序建议增加 CRC 校验。
13. 温度公式为 raw_temp * 175 / 65535 - 45。
14. 湿度公式为 raw_humi * 100 / 65535。
```

## 13. S8 在智能家居项目中的作用

在你的智能家居系统中，S8 主要用于检测环境温度和湿度。其中温度值可以作为风扇控制依据。

### 自动风扇控制逻辑

```text
1. S8 读取当前温度 temperature
2. 如果 temperature > 28℃：
      打开 E2 风扇
3. 如果 temperature <= 28℃：
      关闭 E2 风扇
4. 可将 temperature 显示到 E1 数码管前两位
```

示例逻辑：

```c
float temp, humi;

s8_read(&temp, &humi);

if (temp > 28.0) {
    e2_set_fan_speed(80);
} else {
    e2_set_fan_speed(0);
}
```

## 14. 调试与排错

| 问题         | 可能原因                   | 解决方法                      |
| ---------- | ---------------------- | ------------------------- |
| 扫描不到 S8 地址 | 子板未供电 / 接线错误 / 拨码状态不确定 | 检查电源指示灯，扫描 0x44 和 0x45    |
| 读取数据全为 0   | 未发送测量命令 / 等待时间不足       | 先发送 0x2C0D，等待 100ms 后再读   |
| 温度值异常      | 地址使用错误 / 字节顺序错误 / 公式错误 | 确认使用 7 位还是 8 位地址，检查高低字节顺序 |
| 湿度值超过 100% | 数据解析错误                 | 确认 byte3、byte4 才是湿度数据     |
| 程序卡死       | I2C 无应答                | 增加超时机制，先运行 I2C 扫描         |

---

## 15. 最小读取流程总结

最小可用流程如下：

```text
S8 地址确认：
    0x44 或 0x45

初始化：
    写入 0x30A2
    延时 200ms

读取：
    写入 0x2C0D
    延时 100ms
    读取 6 字节

解析：
    temp_raw = byte0 << 8 | byte1
    humi_raw = byte3 << 8 | byte4

计算：
    temperature = temp_raw * 175 / 65535 - 45
    humidity = humi_raw * 100 / 65535
```
