# C3 子板当前实现说明

本文档只描述仓库里已经落地的 C3 固件实现，不把“计划中能力”写成“已验证能力”。

## 1. 当前状态

- C3 不是 I2C 子板，当前通过 `USART5` 与 U1 主控通信。
- 固件入口在 `main.c` 中执行 `uart_init(USART5)` 后调用 `Wifi_Init()`。
- C3 已接入智能家居主循环，不再是“预留接口”。
- 当前代码已经实现：
  - Wi-Fi 初始化与 AT 指令握手
  - HTTP 上报传感器状态
  - HTTP 轮询后端命令
  - 将云端命令映射为 LED / 风扇 / 模式控制
- 当前实机现状：
  - 代码链路已实现
  - Wi-Fi 实机联调仍未完全打通，本文档描述的是“当前代码行为”，不是“已完成验证结论”

## 2. 当前默认配置

以下默认值直接来自 [c3_wifi.c](/abs/path/d:/xiaomi/Smart_Home/demo/Project/GD32F450Z_BSP/src/c3_wifi.c)：

```c
#define C3_WIFI_SSID         "ozy"
#define C3_WIFI_PASSWORD     "12345678"
#define C3_SERVER_HOST       "192.168.137.1"
#define C3_SERVER_PORT       5000U
#define C3_DEVICE_ID         "smart-home-001"
```

说明：

- `C3_WIFI_SSID` / `C3_WIFI_PASSWORD` 是 C3 目标热点。
- `C3_SERVER_HOST` 是后端服务器地址，不决定 C3 连哪个热点。
- 目前默认目标是 Windows 热点常见网关 `192.168.137.1`。
- 如果电脑热点实际不是这个地址，必须同步修改固件宏定义。

## 3. 固件初始化流程

`Wifi_Init()` 的当前流程如下：

1. 延时 1.5 秒，等待模块稳定。
2. 连续发送 `AT`，最多重试 5 次。
3. 首次启动时发送一次 `AT+RESTORE`，强制恢复模块配置。
4. 发送 `ATE0` 关闭回显。
5. 发送 `AT+CWMODE=1` 进入 Station 模式。
6. 清理历史 Wi-Fi 状态：
   - `AT+CWAUTOCONN=0`
   - `AT+CWRECONNCFG=0,0`
   - `AT+CWQAP`
   - `AT+SYSSTORE=0`
7. 尝试连接目标热点：
   - `AT+CWJAP="ozy","12345678"`
   - 失败后再尝试 `AT+CWJAP_CUR="ozy","12345678"`
8. 通过 `AT+CWJAP?` 校验实际连接的 SSID。
9. 通过 `AT+CIPSTA?` 查询 IP。
10. 优先尝试 `AT+CIPMUX=1`，失败则回退到 `AT+CIPMUX=0`。
11. 启动后立刻对后端执行一次 `GET /api/device/next_cmd` 探测。

这段流程的目的，是尽量避免 ESP32 AT 固件复用旧热点配置并自动连回手机热点。

## 4. 运行时行为

当前主循环在 [main.c](/abs/path/d:/xiaomi/Smart_Home/demo/Project/Application/src/main.c) 中每次循环末尾执行：

```c
Wifi_SendSensorData(...);
Wifi_ReceiveCommand();
wifi_cmd = Wifi_GetPendingCommand();
smart_home_handle_wifi_command(wifi_cmd);
delay_ms(SENSOR_UPDATE_MS);
```

配套参数来自 [smart_home_config.h](/abs/path/d:/xiaomi/Smart_Home/demo/Project/GD32F450Z_BSP/inc/smart_home_config.h)：

```c
#define SENSOR_UPDATE_MS     200U
```

因此当前节奏为：

- 状态上报：每 5 个循环一次，约 1 秒
- 命令轮询：每 5 个循环一次，约 1 秒
- 断线重连：当 `g_wifi_ready == 0` 时，每 25 个循环重试一次，约 5 秒

## 5. 当前 HTTP 协议

### 5.1 上报接口

- 方法：`POST`
- 路径：`/api/device/report`
- Body 示例：

```json
{
  "device_id": "smart-home-001",
  "temperature": 28.50,
  "humidity": 52.30,
  "light": 360,
  "fan": "off",
  "led": "on",
  "mode": "auto"
}
```

### 5.2 命令轮询接口

- 方法：`GET`
- 路径：`/api/device/next_cmd?device_id=smart-home-001`

### 5.3 当前支持的命令

来自 [c3_wifi.h](/abs/path/d:/xiaomi/Smart_Home/demo/Project/GD32F450Z_BSP/inc/c3_wifi.h)：

- `LED_ON`
- `LED_OFF`
- `FAN_ON`
- `FAN_OFF`
- `AUTO_MODE`
- `MANUAL_MODE`
- `READ_STATUS`

其中：

- `LED_ON` / `LED_OFF` 会切到手动模式并控制 LED。
- `FAN_ON` / `FAN_OFF` 会切到手动模式并控制风扇。
- `AUTO_MODE` / `MANUAL_MODE` 只切模式。
- `READ_STATUS` 当前不直接改状态，只依赖下一次周期上报。

## 6. 与主控联动的当前逻辑

当前智能家居控制逻辑为：

- 光照 `< 500 lx` 时，自动模式下打开 LED。
- 风扇为四档温度控制：
  - `>= 24°C` -> `30%`
  - `>= 26°C` -> `50%`
  - `>= 28°C` -> `70%`
  - `>= 30°C` -> `90%`
- 数码管显示规则：
  - 前两位显示温度整数部分
  - 后两位显示 `light / 100`

## 7. 调试输出关键字

当前会从 `USART0` 打印以下关键日志：

- `C3 WIFI TARGET=`
- `C3 RESTORE CONFIG`
- `C3 RESTORE RESP:`
- `C3 AT FAIL:`
- `C3 CWMODE FAIL:`
- `C3 AP INFO:`
- `C3 AP VERIFY FAIL`
- `C3 IP INFO:`
- `C3 CIPMUX=1`
- `C3 CIPMUX=0`
- `C3 SERVER=`
- `C3 PROBE OK`
- `C3 PROBE TCP FAIL`
- `C3 PROBE SEND FAIL`
- `C3 NET LOST`
- `C3 RECONNECT...`

如果 Keil 调试或串口工具看不到这些日志，先检查你监听的是不是 `USART0` 调试口，而不是 `USART5` 的 C3 AT 通道。
