# C3 Wi-Fi + BLE 子板配置与使用说明

## 1. 子板概述

C3 子板是小米 AIoT 实训箱中的 **通用 Wi-Fi + BLE 通信子板**。它集成了 **ESP-WROOM-32-N4 / ESP32 Wi-Fi 模组**，可通过 Wi-Fi 网络和蓝牙实现远程控制与通信，适用于智能家居、智慧园区等物联网场景。

和 E1、E2、S2、S8 不同，C3 **不是 I2C 子板**，它主要通过 **UART 串口 + AT 指令** 进行控制。

---

## 2. 子板硬件组成

| 编号 | 名称        | 数量 | 作用            |
| -- | --------- | -: | ------------- |
| ①  | 串口波特率复位按键 |  1 | 恢复或调整串口通信相关配置 |
| ②  | 电源指示灯     |  1 | 指示子板是否正常供电    |
| ③  | 复位按键      |  1 | 复位 ESP32 模组   |
| ④  | 拨码开关 1    |  1 | 功能拨码          |
| ⑤  | 拨码开关 2    |  1 | 功能拨码          |

资料中的功能框图显示：C3 通过 **UART** 与外部接口通信，核心模块为 ESP32，另一侧通过无线信号实现 Wi-Fi / BLE 通信。

---

## 3. 通信方式

```text
通信协议：UART 串口
控制方式：AT 指令
核心模组：ESP-WROOM-32-N4 / ESP32
无线能力：Wi-Fi + BLE
是否 I2C：否
是否需要 I2C 地址：不需要
```

Agent 必须记住：

```text
C3 不是 I2C 子板。
C3 不需要 I2C 扫描。
C3 不使用 0xXX 形式的 I2C 地址。
C3 应通过串口发送 AT 指令进行控制。
```

---

## 4. C3 是否可以连接到电脑？

### 4.1 可以：通过串口连接电脑进行调试

可以。资料第 5 页展示了电脑端串口调试助手界面，可以看到串口选择、波特率、数据位、校验位、停止位等配置项，并且示例中波特率显示为 **115200**，串口设备类似 **USB-SERIAL CH34x**。这说明 C3 可以通过串口方式与电脑调试工具通信。

典型连接方式：

```text
电脑 USB
   ↓
USB 转 TTL 串口模块 / U 板转接串口
   ↓
C3 子板 UART
```

常见串口参数可以先按资料截图使用：

| 参数  | 推荐值    |
| --- | ------ |
| 波特率 | 115200 |
| 数据位 | 8      |
| 校验位 | None   |
| 停止位 | 1      |
| 流控  | None   |

连接后，可以在串口助手中发送：

```text
AT
```

如果返回：

```text
OK
```

说明串口通信正常。

---

### 4.2 可以：通过 Wi-Fi 连接电脑上的 TCP 服务器

也可以。C3 可以先连接 Wi-Fi，然后连接电脑上运行的 TCP Server。资料中给出了 Wi-Fi 连接和 TCP 发送数据流程，例如 `AT+CWMODE=1`、`AT+CWJAP`、`AT+CIPSTART`、`AT+CIPSEND`。

典型结构：

```text
C3 子板
   ↓ Wi-Fi
路由器 / 手机热点
   ↓ 局域网
电脑 TCP Server
```

电脑端可以运行：

```text
网络调试助手 / NetAssist / Python TCP Server / Flask 后端
```

C3 端通过 AT 指令连接电脑 IP 和端口。

示例：

```text
AT+CWMODE=1
AT+CWJAP="MyRouter","password123"
AT+CIPMUX=1
AT+CIPSTART=0,"TCP","192.168.1.100",8080
AT+CIPSEND=0,5
hello
```

如果发送成功，模块会返回：

```text
SEND OK
```

注意：电脑和 C3 必须在同一个局域网内，电脑防火墙要允许对应端口访问。

---

### 4.3 蓝牙连接电脑：理论可行，但不如手机稳定

资料中蓝牙部分主要演示的是手机连接 C3，流程包括初始化蓝牙、设置蓝牙名称、开启扫描模式、启动 SPP 服务，并等待设备连接。

理论上，如果电脑蓝牙支持 **SPP 串口服务**，可以把 C3 当作蓝牙串口设备连接。但 Windows 上不一定默认好用，可能需要配对后生成虚拟 COM 口。

所以建议优先使用：

```text
串口连接电脑：用于调试 AT 指令
Wi-Fi 连接电脑：用于项目数据上传和远程控制
蓝牙连接手机：用于演示移动端控制
```

---

## 5. C3 的 AT 指令使用流程

## 5.1 基础测试

连接串口后，先测试：

```text
AT
```

期望返回：

```text
OK
```

如果没有返回，需要检查：

```text
1. 串口号是否正确
2. 波特率是否为 115200
3. TX/RX 是否接反
4. GND 是否共地
5. C3 是否正常供电
6. 是否需要按复位键
```

---

## 5.2 Wi-Fi 模式配置

设置为 Station 模式：

```text
AT+CWMODE=1
```

返回：

```text
OK
```

连接 Wi-Fi：

```text
AT+CWJAP="WiFi名称","WiFi密码"
```

成功返回：

```text
WIFI CONNECTED
WIFI GOT IP
OK
```

---

## 5.3 TCP 连接电脑服务器

开启多连接：

```text
AT+CIPMUX=1
```

连接电脑 TCP Server：

```text
AT+CIPSTART=0,"TCP","电脑IP地址",端口
```

例如：

```text
AT+CIPSTART=0,"TCP","192.168.1.100",8080
```

成功返回：

```text
CONNECT
OK
```

发送数据：

```text
AT+CIPSEND=0,5
```

返回提示符：

```text
>
```

然后输入：

```text
hello
```

成功返回：

```text
SEND OK
```

---

## 5.4 蓝牙 SPP 使用流程

初始化蓝牙：

```text
AT+BTINIT=1
```

初始化 SPP：

```text
AT+BTSPPINIT=2
```

设置蓝牙名称：

```text
AT+BTNAME="ESP32_SPP"
```

设置扫描模式：

```text
AT+BTSCANMODE=2
```

启动 SPP：

```text
AT+BTSPPSTART
```

等待手机或其他设备连接。连接后模块可能返回类似：

```text
+BTSPPCONN:0,"e0:24:81:47:90:bc"
```

进入透传发送：

```text
AT+BTSPPSEND
```

之后可以直接输入数据：

```text
HelloWorld
```

退出透传：

```text
+++
```

断开连接：

```text
AT+BTSPPDISCONN=0
```

# 6. 在智能家居项目中的使用方式

C3 在你的智能家居项目中主要适合做 **联网通信模块**，而不是直接控制灯、风扇、传感器。

推荐分工如下：

| 子板 | 作用                         |
| -- | -------------------------- |
| S2 | 采集光照                       |
| S8 | 采集温湿度                      |
| E1 | 控制 LED 和数码管显示              |
| E2 | 控制风扇                       |
| C3 | Wi-Fi / 蓝牙通信，上传数据或接收远程控制命令 |

---

## 7. C3 与电脑连接方案推荐

### 方案一：电脑串口调试 C3

适合实验初期。

```text
电脑串口助手
   ↓ USB 串口
C3 子板
```

用途：

```text
1. 测试 AT 指令
2. 配置 Wi-Fi
3. 查看模块返回信息
4. 验证 C3 是否正常工作
```

推荐工具：

```text
串口调试助手
MobaXterm 串口模式
XCOM
SSCOM
PuTTY Serial
```

---

### 方案二：C3 通过 Wi-Fi 连接电脑 TCP 服务

适合项目演示。

```text
S2 / S8 采集数据
   ↓
U1 主控处理
   ↓ UART
C3 子板
   ↓ Wi-Fi
电脑 TCP Server / Flask 后端
```

电脑端可以开一个 TCP Server，C3 发送数据：

```json
{
  "temperature": 28.5,
  "humidity": 52.3,
  "light": 360,
  "fan": "on",
  "led": "off"
}
```

这样就可以在电脑上展示智能家居状态。

---

### 方案三：C3 连接手机蓝牙

适合移动端控制演示。

```text
手机蓝牙
   ↓
C3 BLE / SPP
   ↓ UART
U1 主控
   ↓
控制 E1 / E2
```

手机可以发送：

```text
LIGHT_ON
FAN_ON
AUTO_MODE
ENERGY_SAVE
```

主控收到后执行对应动作。

# 8. Agent 知识库配置示例

可以写入 `boards.yaml`：

```yaml
C3:
  name: "通用 Wi-Fi + BLE 子板"
  type: "communication"
  protocol: "uart"
  chip: "ESP-WROOM-32-N4 / ESP32"
  is_i2c: false
  i2c_address: null

  hardware:
    power_led: true
    reset_button: true
    baudrate_reset_button: true
    dip_switches: 2

  uart:
    default_baudrate: 115200
    data_bits: 8
    parity: "None"
    stop_bits: 1
    flow_control: "None"

  wifi:
    mode_command: "AT+CWMODE=1"
    connect_command: "AT+CWJAP=\"<ssid>\",\"<password>\""
    tcp_multi_connection: "AT+CIPMUX=1"
    tcp_connect: "AT+CIPSTART=0,\"TCP\",\"<host>\",<port>"
    tcp_send: "AT+CIPSEND=0,<length>"

  bluetooth_spp:
    init: "AT+BTINIT=1"
    spp_init: "AT+BTSPPINIT=2"
    set_name: "AT+BTNAME=\"ESP32_SPP\""
    scan_mode: "AT+BTSCANMODE=2"
    start: "AT+BTSPPSTART"
    send_mode: "AT+BTSPPSEND"
    exit_passthrough: "+++"
    disconnect: "AT+BTSPPDISCONN=0"

  agent_actions:
    - "test_uart_at"
    - "connect_wifi"
    - "connect_tcp_server"
    - "send_tcp_data"
    - "init_bluetooth_spp"
    - "send_bluetooth_data"
```

# 9. Agent 编程规则

Agent 在处理 C3 子板时必须遵守：

```text
1. C3 是通信子板，不是传感器子板，也不是执行器子板。
2. C3 使用 UART 串口通信，不使用 I2C。
3. C3 不需要 I2C 地址，不能使用 I2C 扫描查找。
4. C3 通过 AT 指令控制 ESP32 模组。
5. 连接电脑有两种主要方式：
   - 串口连接电脑：用于调试 AT 指令；
   - Wi-Fi 连接电脑 TCP Server：用于数据上传和远程控制。
6. 蓝牙连接电脑取决于电脑蓝牙 SPP 支持情况，不能保证所有电脑都可直接使用。
7. 生成代码前必须确认：
   - 串口号；
   - 波特率；
   - Wi-Fi 名称和密码；
   - 电脑 IP；
   - TCP 端口；
   - 是否需要透传模式。
8. 如果用户说“连接电脑”，Agent 要先判断是：
   - 通过串口连电脑调试；
   - 通过 Wi-Fi 连电脑服务；
   - 通过蓝牙连电脑。
```

# 10. 最小可用测试流程

## 10.1 串口测试

```text
1. C3 插到 U1 或通过 USB-TTL 连接电脑
2. 打开串口助手
3. 选择正确 COM 口
4. 设置波特率 115200
5. 发送 AT
6. 收到 OK，说明 C3 串口通信正常
```

## 10.2 Wi-Fi + TCP 测试

电脑端：

```text
1. 打开网络调试助手
2. 设置为 TCP Server
3. 监听端口 8080
4. 查看电脑局域网 IP，例如 192.168.1.100
```

C3 端：

```text
AT+CWMODE=1
AT+CWJAP="你的WiFi","你的密码"
AT+CIPMUX=1
AT+CIPSTART=0,"TCP","192.168.1.100",8080
AT+CIPSEND=0,11
hello world
```

电脑端能收到 `hello world`，说明 C3 已成功连接电脑 TCP 服务。

# 11. 调试问题排查

| 问题         | 可能原因                     | 解决方法                             |
| ---------- | ------------------------ | -------------------------------- |
| 串口没有返回 OK  | 波特率错误 / 串口选错 / TX RX 接反  | 检查 COM 口，使用 115200，交换 TX/RX      |
| 返回乱码       | 波特率不匹配                   | 尝试 115200，或按波特率复位键               |
| Wi-Fi 连接失败 | SSID 或密码错误 / 信号弱         | 检查 Wi-Fi 名称密码，靠近路由器              |
| TCP 连接失败   | 电脑 IP 错误 / 防火墙拦截 / 端口未监听 | 关闭或放行防火墙，确认 TCP Server 已启动       |
| 蓝牙找不到设备    | 未启动 SPP / 未设置扫描模式        | 执行 `AT+BTINIT=1`、`AT+BTSPPSTART` |
| 电脑蓝牙无法连接   | Windows 未创建蓝牙串口服务        | 优先用手机蓝牙或 Wi-Fi 方案                |

## 12. 总结

C3 子板的定位是：

```text
C3 = ESP32 通信模块 + UART AT 指令控制 + Wi-Fi/BLE 无线连接
```

它可以连接电脑，推荐两种方式：

```text
方式一：电脑通过串口直接调试 C3
方式二：C3 通过 Wi-Fi 连接电脑 TCP Server
```

在你的智能家居项目里，最推荐使用：

```text
S2/S8 负责采集数据
E1/E2 负责显示和执行
C3 负责把数据发送到电脑端 Flask/TCP 服务，或者接收电脑端控制命令
```
