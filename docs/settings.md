# 智能家居系统所需子板配置与硬件地址汇总

## 1. 子板总体配置

本系统采用 **U1 作为主控子板**，通过 I2C 总线连接 S1、S2、S8、E1、E2 等功能子板，并预留 C3 Wi-Fi 子板作为后续联网与 LLM 自然语言控制接口。

```text
U1 主控子板
├── S2 光照传感器：采集光照值
├── S8 温湿度传感器：采集温度值
├── E1 LED + 数码管：控制 LED，显示温度和光照
├── E2 风扇子板：控制风扇开关 / 转速
├── S1 按键子板：切换自动 / 手动模式，手动控制设备
└── C3 Wi-Fi 子板：预留联网和 LLM 接口
```

# 3. 推荐地址配置方案

为了避免地址冲突，建议本项目采用以下地址配置：

| 子板            | 推荐地址 | 说明               |
| --------------- | -------- | ------------------ |
| S2 光照传感器   | `0x46`   | 默认使用第一组地址 |
| S8 温湿度传感器 | `0x88`   | 默认使用第一组地址 |
| E1 LED          | `0xC0`   | 控制 RGB 三色灯    |
| E1 数码管       | `0xE0`   | 控制四位数码管     |
| E2 风扇         | `0xC8`   | 控制风扇 PWM       |
| S1 按键         | `0xE8`   | 读取按键输入       |
| C3 Wi-Fi        | UART     | 后续扩展使用       |

推荐最终地址表：

```c
#define S2_LIGHT_ADDR        0x46
#define S8_TEMP_HUMI_ADDR    0x88

#define E1_LED_ADDR          0xC0
#define E1_DISPLAY_ADDR      0xE0

#define E2_FAN_ADDR          0xC8
#define S1_KEY_ADDR          0xE8
```

# 4. 各子板配置说明

## 4.1 U1 主控子板配置

| 项目     | 配置                                       |
| -------- | ------------------------------------------ |
| 子板名称 | U1 主控子板                                |
| MCU      | GD32470ZKT6                                |
| 内核     | Arm Cortex-M4                              |
| 主频     | 最高 240MHz                                |
| FLASH    | 3072KB                                     |
| RAM      | 256KB                                      |
| 主要作用 | 传感器采集、执行器控制、显示控制、模式切换 |
| 使用外设 | I2C、UART、GPIO、定时器                    |

U1 在本系统中负责统一调度：

```text
1. 通过 I2C 读取 S2 光照值
2. 通过 I2C 读取 S8 温度值
3. 通过 I2C 读取 S1 按键值
4. 通过 I2C 控制 E1 LED 和数码管
5. 通过 I2C 控制 E2 风扇
6. 通过 UART 预留 C3 Wi-Fi 通信接口
```

## 4.2 S2 光照传感器子板配置

| 项目         | 配置              |
| ------------ | ----------------- |
| 子板名称     | S2 光照传感器子板 |
| 核心器件     | BH1750            |
| 功能         | 采集环境光照强度  |
| 通信方式     | I2C               |
| 地址选择方式 | 拨码开关          |
| 可选地址     | `0x46` / `0xB8`   |
| 推荐地址     | `0x46`            |
| 检测范围     | 1–65535 lx        |
| 供电转换     | 5V 转 3.3V        |

### 常用命令

| 命令   | 功能             |
| ------ | ---------------- |
| `0x00` | 断电             |
| `0x01` | 上电             |
| `0x07` | 复位             |
| `0x10` | 连续高精度模式   |
| `0x11` | 连续高精度模式 2 |
| `0x13` | 连续低精度模式   |
| `0x20` | 单次高精度模式   |
| `0x21` | 单次高精度模式 2 |
| `0x23` | 单次低精度模式   |

### 本项目使用方式

```text
S2 读取光照值 lux
        ↓
U1 判断 lux 是否小于光照阈值
        ↓
光照弱：打开 E1 LED
光照强：关闭 E1 LED
        ↓
数码管后两位显示简略光照值
```

## 4.3 S8 温湿度传感器子板配置

| 项目         | 配置                |
| ------------ | ------------------- |
| 子板名称     | S8 温湿度传感器子板 |
| 核心器件     | SHT35-DIS-B         |
| 功能         | 采集温度和湿度      |
| 通信方式     | I2C                 |
| 地址选择方式 | 拨码开关            |
| 可选地址     | `0x88` / `0x8A`     |
| 推荐地址     | `0x88`              |
| 温度范围     | -40℃ 到 125℃        |
| 温度精度     | ±0.1℃               |
| 湿度范围     | 0%RH 到 100%RH      |
| 湿度精度     | ±1.5%RH             |

### 常用命令

| 命令     | 功能         |
| -------- | ------------ |
| `0x30A2` | 软复位 SHT35 |
| `0x2C0D` | 连续读模式   |

### 读取数据格式

S8 一次读取 6 个字节：

```text
Byte0：温度高字节
Byte1：温度低字节
Byte2：温度 CRC
Byte3：湿度高字节
Byte4：湿度低字节
Byte5：湿度 CRC
```

### 数据计算公式

```c
temperature_raw = (temp_high << 8) + temp_low;
temperature = temperature_raw * 175 / 65535 - 45;

humidity_raw = (humi_high << 8) + humi_low;
humidity = humidity_raw * 100 / 65535;
```

### 本项目使用方式

```text
S8 读取温度值
        ↓
U1 判断温度是否大于阈值
        ↓
温度高：打开 E2 风扇
温度低：关闭 E2 风扇
        ↓
数码管前两位显示温度
```

## 4.4 E1 LED + 数码管子板配置

E1 子板内部包含两个主要部分：

```text
1. RGB LED 控制部分：PCA9685
2. 四位数码管显示部分：HT16K33
```

因此 E1 子板在程序中需要分别配置 **LED 地址**和**数码管地址**。

------

## 4.4.1 E1 LED 部分配置

| 项目     | 配置                              |
| -------- | --------------------------------- |
| 子板功能 | RGB LED 控制                      |
| 驱动芯片 | PCA9685                           |
| 控制方式 | PWM                               |
| 通信方式 | I2C                               |
| 可选地址 | `0xC0` / `0xC2` / `0xC4` / `0xC6` |
| 推荐地址 | `0xC0`                            |

### LED 控制寄存器

| 寄存器        | 名称                     | 功能              |
| ------------- | ------------------------ | ----------------- |
| `0x00`        | MODE1                    | 模式控制          |
| `0x06`–`0x09` | LED0_ON / LED0_OFF       | LED0 PWM 输出     |
| `0x0A`–`0x0D` | LED1_ON / LED1_OFF       | LED1 PWM 输出     |
| `0x0E`–`0x11` | LED2_ON / LED2_OFF       | LED2 PWM 输出     |
| `0xFA`–`0xFD` | ALL_LED_ON / ALL_LED_OFF | 全部 LED 输出控制 |

### 初始化步骤

```text
1. 向寄存器 0x00 写入 0x00，初始化 PCA9685
2. 向 0xFA–0xFD 写入 0x00，关闭全部输出
3. 向 0x06–0x11 写入 PWM 值，控制 RGB 亮度
```

### 本项目 LED 控制

```text
自动模式：
    光照小于阈值 → 打开 LED
    光照大于等于阈值 → 关闭 LED

手动模式：
    K2 控制 LED 开关
```

## 4.4.2 E1 数码管部分配置

| 项目       | 配置                              |
| ---------- | --------------------------------- |
| 子板功能   | 四位数码管显示                    |
| 驱动芯片   | HT16K33                           |
| 数码管类型 | 4 位数码管                        |
| 通信方式   | I2C                               |
| 可选地址   | `0xE0` / `0xE2` / `0xE4` / `0xE6` |
| 推荐地址   | `0xE0`                            |

### 数码管控制寄存器

| 寄存器 / 命令 | 功能               |
| ------------- | ------------------ |
| `0x02`–`0x03` | COM1，第一位数码管 |
| `0x04`–`0x05` | COM2，第二位数码管 |
| `0x06`–`0x07` | COM3，第三位数码管 |
| `0x08`–`0x09` | COM4，第四位数码管 |
| `0x21`        | 开启系统振荡器     |
| `0xA0`        | ROW / INT 设置     |
| `0x81`        | 打开显示           |
| `0x80`        | 关闭显示           |

### 初始化步骤

```text
1. 发送命令 0x21，开启系统振荡器
2. 发送命令 0xA0，设置 ROW / INT
3. 向 0x02–0x09 写入 0x00，清空显示
4. 发送命令 0x81，打开显示
```

### 数码管显示分配

```text
COM1 COM2 | COM3 COM4
 温度两位 | 光照两位
```

例如：

```text
温度 26℃，光照 1200 lx
光照简略值 = 1200 / 100 = 12
显示内容 = 2612
```

## 4.5 E2 风扇子板配置

| 项目           | 配置                              |
| -------------- | --------------------------------- |
| 子板名称       | E2 风扇子板                       |
| 核心芯片       | PCA9685                           |
| 风扇电机       | RC300C 微型直流电机               |
| 功能           | PWM 控制风扇转速                  |
| 通信方式       | I2C                               |
| 地址选择方式   | 拨码开关                          |
| 可选地址       | `0xC8` / `0xCA` / `0xCC` / `0xCE` |
| 推荐地址       | `0xC8`                            |
| PWM 频率范围   | 40–1000Hz                         |
| PWM 占空比范围 | 0%–100%                           |

### 风扇控制寄存器

| 寄存器        | 名称                     | 功能              |
| ------------- | ------------------------ | ----------------- |
| `0x00`        | MODE1                    | 模式控制          |
| `0x06`–`0x09` | LED0_ON / LED0_OFF       | 风扇 PWM 输出控制 |
| `0xFA`–`0xFD` | ALL_LED_ON / ALL_LED_OFF | 全部输出控制      |

### 初始化步骤

```text
1. 向寄存器 0x00 写入 0x00，初始化 PCA9685
2. 向 0xFA–0xFD 写入 0x00，关闭输出
3. 向 0x06–0x09 写入 PWM 值，控制风扇转速
```

### 本项目风扇控制

```text
自动模式：
    温度 >= 温度阈值 → 打开风扇
    温度 < 温度阈值 → 关闭风扇

手动模式：
    K3 控制风扇开关
```

建议初期只做两档：

| 状态 | PWM 占空比 |
| ---- | ---------- |
| 关闭 | 0%         |
| 打开 | 60% 或 80% |

## 4.6 S1 按键子板配置

| 项目         | 配置                              |
| ------------ | --------------------------------- |
| 子板名称     | S1 按键子板                       |
| 核心芯片     | HT16K33                           |
| 按键数量     | 12 个                             |
| 键盘结构     | 3×4 矩阵键盘                      |
| 通信方式     | I2C                               |
| 地址选择方式 | 拨码开关                          |
| 可选地址     | `0xE8` / `0xEA` / `0xEC` / `0xEE` |
| 推荐地址     | `0xE8`                            |

### 按键相关寄存器

| 寄存器 / 命令 | 功能                             |
| ------------- | -------------------------------- |
| `0x21`        | 开启系统振荡器，进入正常工作模式 |
| `0x20`        | 关闭系统振荡器，进入待机模式     |
| `0x40`        | 读取第一列按键值                 |
| `0x42`        | 读取第二列按键值                 |
| `0x44`        | 读取第三列按键值                 |

### 本项目按键分配

只使用 3 个按键：

| 按键 | 功能                | 有效模式                 |
| ---- | ------------------- | ------------------------ |
| K1   | 自动 / 手动模式切换 | 自动模式、手动模式均有效 |
| K2   | LED 开 / 关         | 仅手动模式有效           |
| K3   | 风扇开 / 关         | 仅手动模式有效           |

### 按键逻辑

```text
K1：
    自动模式 ↔ 手动模式

K2：
    手动模式下，LED 开 ↔ LED 关

K3：
    手动模式下，风扇开 ↔ 风扇关
```

## 4.7 C3 Wi-Fi 子板配置

| 项目         | 配置                |
| ------------ | ------------------- |
| 子板名称     | C3 Wi-Fi / BLE 子板 |
| 核心模组     | ESP32               |
| 功能         | Wi-Fi / 蓝牙通信    |
| 通信方式     | UART                |
| 当前用途     | 预留后续 LLM 接口   |
| 是否当前实现 | 暂不实现            |

### 工作模式

| 模式             | 说明                                                    |
| ---------------- | ------------------------------------------------------- |
| Passthrough 模式 | Pogo Pin 上下连通，ESP32 旁路，只连接 UART1             |
| Master 模式      | UART 和 I2C 都连接到 ESP32，默认 OFF 状态为 Master 模式 |

### 后续扩展用途

```text
U1 采集温度、湿度、光照和设备状态
        ↓
通过 UART 发送给 C3 Wi-Fi 子板
        ↓
C3 上传到后端服务
        ↓
后端调用 LLM
        ↓
返回控制命令
        ↓
U1 执行 LED / 风扇 / 模式控制
```

### 可预留接口

```c
void Wifi_Init(void);
void Wifi_SendSensorData(float temp, float humi, uint16_t light);
void Wifi_ReceiveCommand(void);
```

# 7. 地址冲突检查

本项目各子板推荐地址如下：

```text
S2：0x46
S8：0x88
E1 LED：0xC0
E1 数码管：0xE0
E2 风扇：0xC8
S1 按键：0xE8
```

这些地址之间不冲突，可以挂载在同一条 I2C 总线上。

需要注意的是：

```text
E1 子板内部有两个 I2C 设备：
1. PCA9685：控制 RGB LED，地址 0xC0
2. HT16K33：控制数码管，地址 0xE0
```

所以 E1 在软件上应拆成两个驱动模块：

```text
e1_led.c
e1_display.c
```

# 8. 最终推荐硬件配置清单

| 类型     | 子板 | 数量 | 推荐地址 / 接口             | 项目用途            |
| -------- | ---- | ---- | --------------------------- | ------------------- |
| 主控     | U1   | 1    | 主控                        | 系统核心            |
| 传感器   | S2   | 1    | `0x46`                      | 光照采集            |
| 传感器   | S8   | 1    | `0x88`                      | 温度采集            |
| 执行器   | E1   | 1    | LED：`0xC0`，数码管：`0xE0` | LED 控制 + 数据显示 |
| 执行器   | E2   | 1    | `0xC8`                      | 风扇控制            |
| 输入     | S1   | 1    | `0xE8`                      | 三按键控制          |
| 通信扩展 | C3   | 1    | UART                        | 后续联网和 LLM 扩展 |

最终硬件结构可以写为：

```text
U1 主控通过 I2C 总线分别连接 S2、S8、E1、E2、S1 子板，其中 S2 地址配置为 0x46，S8 地址配置为 0x88，E1 子板中的 LED 驱动 PCA9685 地址配置为 0xC0，数码管驱动 HT16K33 地址配置为 0xE0，E2 风扇 PCA9685 地址配置为 0xC8，S1 按键 HT16K33 地址配置为 0xE8。C3 Wi-Fi 子板通过 UART 接口预留，用于后续联网和大语言模型自然语言控制扩展。
```
